# Информационная система службы технической поддержки

## Наименование проекта

Официальное наименование проекта: *Автоматизированная система учета обращений службы технической поддержки отдела информатизации ГБПОУ ЯНАО «Ямальский многопрофильный колледж»*.

Программный продукт выполнен в формате десктоп-приложения с использованием языка Python и библиотек Tkinter и Peewee.

## Этапы реализации
* Построение диаграмм **прецедентов** и **классов**.
* Проектирование структуры и наполнение базы данных MySQL.
* Формирование архитектуры проекта (модели, контроллеры, представления).
* Реализация требуемого функционала:
  * разграничение доступа (RBAC),
  * обработка пользовательских обращений,
  * накопление справочной информации (база знаний),
  * администрирование учетных записей,
  * панель с базовой аналитикой.

## Схема базы данных

В качестве системы управления базами данных применяется **MySQL**, взаимодействие осуществляется через модуль `Connect/connect.py`, описание сущностей выполнено с помощью ORM Peewee.

### Users (учетные записи)
* `id` – уникальный идентификатор.
* `name` – ФИО пользователя.
* `role_id` – ссылка на таблицу `Role` (определяет права доступа).
* `login` – уникальное имя для входа.
* `password` – пароль в зашифрованном виде (bcrypt).

### Role (уровни доступа)
* `id` – идентификатор.
* `name` – наименование роли.
* Предопределенные роли:
  * `USER` – обычный пользователь,
  * `SPECIALIST` – технический специалист,
  * `ADMIN` – администратор системы.

### Ticket (обращения)
* `id` – идентификатор заявки.
* `title` – краткий заголовок (включает отметку о приоритете).
* `description` – детальное описание проблемы.
* `status_id` – ссылка на `Status` (текущее состояние).
* `created_at` – дата и время регистрации.
* `updated_at` – дата и время последней корректировки.
* `user_id` – ссылка на `Users` (кто создал).
* `executor_id` – ссылка на `Users` (ответственный специалист).
* `category_id` – ссылка на `Category` (тематика).

### Category (тематики обращений)
* `id` – идентификатор.
* `title` – уникальное название категории.
* `description` – описание.
* Предустановленный перечень (проблемы с оборудованием, софтом, доступом и др.) формируется в `create_table.py`.

### Status (состояния заявки)
* `id` – идентификатор.
* `name` – название статуса.
* Фиксированный набор:
  * `NEW` – «Новое»,
  * `IN_PROGRESS` – «Выполняется»,
  * `DONE` – «Завершенно».

### Comment (комментарии)
* `id` – идентификатор.
* `description` – текст комментария.
* `user_id` – ссылка на `Users` (автор).
* `ticket_id` – ссылка на `Ticket` (связанная заявка).
* `type_id` – ссылка на `Type` (видимость комментария).

### ArticleBase (справочные материалы)
* `id` – идентификатор.
* `title` – заголовок инструкции.
* `description` – содержание.
* `ticket_id` – ссылка на `Ticket` (обращение, послужившее основой).
* `executor_id` – ссылка на `Users` (сотрудник, подготовивший материал).

### Type (типы комментариев)
* `id` – идентификатор.
* `name` – наименование.
* Варианты:
  * `PUBLIC` – общедоступный,
  * `INTERNAL` – служебный.

## Используемые технологий

* **Python** – язык разработки.
* **Tkinter** – фреймворк для интерфейса.
* **MySQL** – база данных.
* **Peewee** – объектно-реляционное отображение.
* **mysqlclient** – драйверы.
* **bcrypt** – шифрование паролей.

## Развертывание и запуск

Предварительно требуется создать базу данных.

1. Установка зависимостей из `packages.txt`:

   ```bash
   pip install -r packages.txt
   ```

2. Настройка параметров подключения в `Connect/connect.py` (имя БД, учетные данные).

3. Формирование таблиц и наполнение справочников:

   ```bash
   python Models/create_table.py
   ```

4. Старт приложения:

   ```bash
   python main.py
   ```

   После запуска отображается окно входа, доступен переход к регистрации.

## Возможности системы

### Управление доступом

Контроллер: `UserController` (`Controllers/user_controller.py`).

* Создание учетной записи с шифрованием пароля.
* Вход по логину/паролю.
* Права в зависимости от роли:
  * **Пользователь** – создание и просмотр собственных заявок, доступ к базе знаний.
  * **Специалист** – работа с назначенными обращениями, использование базы знаний.
  * **Администратор** – полное управление заявками, учетными записями, аналитика.

### Обработка заявок

Контроллер: `TicketController` (`Controllers/ticket_controller.py`), интерфейсы `main_view.py`, `create_ticket_view.py`, `ticket_detail_view.py`, `assign_ticket_view.py`.

* Создание обращения с указанием темы, описания, категории и приоритета.
* Фильтрация списка:
  * пользователь видит только свои,
  * специалист – закрепленные за ним,
  * администратор – все.
* Назначение исполнителя (доступно администратору).
* Автоматическое изменение статуса: при назначении – «В работе», при закрытии – «Завершено».

### Комментирование

Контроллер: `CommentController` (`Controllers/comment_controller.py`), окно `ticket_detail_view.py`.

* Добавление комментариев к заявкам.
* Публичные комментарии видны всем участникам, служебные – только специалистам и администратору.

### База знаний

Контроллер: `ArticleController` (`Controllers/article_controller.py`), окна `article_create_view.py`, `kb_view.py`.

* Формирование статьи по закрытому обращению.
* Поиск материалов по заголовку.
* Доступ к базе знаний для всех ролей.

### Администрирование

Окно: `users_view.py` (доступно администратору).

* Просмотр перечня пользователей.
* Удаление учетных записей.

### Аналитика

Контроллер: `StatisticsController` (`Controllers/statistics_controller.py`), окно `stats_view.py`.

* Отображение:
  * общего числа заявок,
  * количества выполненных,
  * количества находящихся в работе.
